Fabric helpers¶

Usage¶

A typical fabfile.py using fusionbox.fabric_helpers looks like:

from fusionbox.fabric_helpers import *

env.roledefs = {
        'dev': ['dev.fusionbox.com'],
        }

env.project_name = 'foobar'
env.short_name = 'fb'

stage = roles('dev')(stage)
fusionbox.fabric_helpers.deploy()[source]¶

Like stage, but always migrates, pips, and uses the live branch

fusionbox.fabric_helpers.files_changed(version, files)[source]¶

Between version and HEAD, has anything in files changed?

fusionbox.fabric_helpers.stage(pip=False, migrate=False, syncdb=False, branch=None)[source]¶

stage will update the remote git version to your local HEAD, collectstatic, migrate and update pip if necessary.

Set env.project_name and env.short_name appropriately to use. env.tld defaults to .com

fusionbox.fabric_helpers.update_git(branch)[source]¶

Updates the remote git repo to branch. Returns the previous remote git version.
